# Moral Summarization

Code for the *News is More than a Collection of Facts: Moral Frame Preserving News Summarization* COLM2025 paper. Currently available as a [pre-print](https://arxiv.org/abs/2504.00657).

This repository has been extended to support **Few-Shot Prompting** (supporting both Direct/Simple and Chain-of-Thought pipelines) which bridges the moral coverage and political ideology stance.

---

## 1. Environment Setup

To ensure all modules run correctly on your local machine, set up the dedicated Conda environment:

conda create -n moral_env python=3.10

pip install pandas scikit-learn sentence-transformers nltk inflect torch

pip install -e . --no-deps

## 2. Dynamic Few-Shot Exemplar Selection Strategy

The pipeline implements a mathematically grounded exemplar selector inside `moral_summarization/select_exemplars.py`. For any target article $T$, the pipeline dynamically selects a triplet of training exemplars $E^* = \{e_L, e_C, e_R\}$ by solving the following optimization objective:

$$E^* = \arg\max_{E} \left[ \alpha \sum_{e \in E} \text{Sim}(e, T) + \beta \cdot \text{Diversity}(E) \right]$$

### Key Mathematical Pillars:
1. **Political Ideology Equilibrium (Equity Axis)**: Selections are strictly constrained to contain exactly one Left ($I = -1$), one Center ($I = 0$), and one Right ($I = 1$) training exemplar. This structural constraint mathematically guarantees:
   $$\text{Bias}(E) = |I(e_L) + I(e_C) + I(e_R)| = |-1 + 0 + 1| = 0$$
2. **Semantic Similarity ($\text{Sim}$)**: Employs cosine similarity over dense text embeddings generated by `sentence-transformers` (`all-MiniLM-L6-v2`) to capture context and structure.
3. **MFT Category Diversity ($\text{Diversity}$)**: Rewards selections covering a wide range of moral foundations. It measures the sum of pairwise cosine distances of the 10-dimensional MFT frequency vectors between the selected exemplars to penalize redundancy and promote comprehensive moral coverage:
   $$\text{Diversity}(E) = 3.0 - \left[ \text{Sim}_{\text{MFT}}(e_L, e_C) + \text{Sim}_{\text{MFT}}(e_L, e_R) + \text{Sim}_{\text{MFT}}(e_C, e_R) \right]$$

## 4. Prompt Method Mapping

The naming convention of the prompting methods is different from the one used in the paper. Here is the mapping:

| Code / CLI Name | Paper Equivalent | Description |
| :--- | :--- | :--- |
| `vanilla` | Plain | Simple standard summary prompt |
| `simple` | Direct | Prompt instructing moral framing preservation |
| `cot` | CoT | 0-shot Chain-of-Thought word extraction then summary |
| `oracle` | Oracle | Injects ground truth moral words from human annotation |
| `class` | Class | Injects classifier predicted moral words |
| `simple_fewshot` | Balanced Few-Shot Direct | Injects 3 balanced Left/Center/Right exemplars showing input and summary |
| `cot_fewshot` | Balanced Few-Shot CoT | Injects 3 balanced Left/Center/Right exemplars showing input, CoT, and summary |
| `simple_fewshot_mft` | Balanced Few-Shot Direct + MFT | Prepend 3 balanced Left/Center/Right exemplars with explicitly active MFT foundation axes |
| `cot_fewshot_mft` | Balanced Few-Shot CoT + MFT | Prepend 3 balanced Left/Center/Right exemplars with active MFT axes and reasoning summaries |

## 5. Generated Prompt Database Breakdown

Our prompt pipeline contains a massive database of prompts generated for all **400 test set articles** across the EMONA corpora. With **8 distinct prompting strategies** generated per article, the pipeline has successfully produced exactly **3,200 formatted prompt files** in `results/test_prompts/`.

### 1. Test Set Stance Distribution
* **AllSides (180 Articles)**: Explicit Left (`l`), Center (`c`), and Right (`r`) political alignment (60 articles per stance bucket).
* **BASIL (150 Articles)**: Explicit Left (`hpo` / Huffington Post), Center (`nyt` / New York Times), and Right (`fox` / Fox News) sources (50 articles per source bucket).
* **MPQA (70 Articles)**: Non-partisan text files used as baseline moral summaries.

### 2. Prompt Strategies Generated
For each of the 400 target articles, the following 8 text files are constructed and stored within its dedicated subfolder (e.g., `results/test_prompts/allsides/allsides_elections_r_5/`):
1. `vanilla_prompt.txt`: Plain summary baseline.
2. `simple_prompt.txt`: Direct zero-shot prompt instructing moral preservation.
3. `cot_prompt.txt`: 0-shot Chain-of-Thought reasoning prompt.
4. `oracle_prompt.txt`: Zero-shot prompt injecting ground truth annotated moral words.
5. `simple_fewshot_prompt.txt`: 3-shot politically balanced direct prompt.
6. `cot_fewshot_prompt.txt`: 3-shot politically balanced CoT reasoning prompt.
7. `simple_fewshot_mft_prompt.txt`: 3-shot politically balanced direct prompt with active MFT dimensions injected.
8. `cot_fewshot_mft_prompt.txt`: 3-shot politically balanced CoT prompt with active MFT dimensions injected.

## 6. Usage

### 1. Dynamic Few-Shot Prompt Generation
To dynamically select balanced exemplars and generate the full database of test prompts, run:

```bash
python generate_prompts.py \
  --prompt-dir results/test_prompts \
  --predictions-path results/llama-8b-predictions-test-set.csv \
  --num-shots 3
```

### 2. Pipeline Training, Cross-Validation, & Inference
1. **Train the Classifier**:
   `python training.py --hf-model-folder <location-of-models> --config-file token_labeling.yaml`
2. **Hyperparameter Tuning**:
   `python hyperparameter_testing.py --hf-model-folder <location-of-models> --config-file token_labeling.yaml`
3. **Execute Summarization Inference**:
   `python prompting.py --config-file peft_config.yaml --hf-model-folder <location-of-models> --prompt-dir results/test_prompts`
4. **Automated Evaluation**:
   `python evaluate.py --llama --QaFactEval --results-dir results/test_prompts`
